A scene is an object near the top of the Cadejo tree whose primary duty is to distribute
incoming MIDI events to appropriate channels. Secondary duties are related to default
property values and maintenance of alternate tuning tables.

MIDI

To use MIDI select an available MIDI device from those listed on the left
and click 'Connect'. Once a MIDI input device has been established it is not possible to change it.

Currently Overtone uses the Java JVM for MIDI access. Unfortunalty out of the box Java does not play well within
a typical Linux MIDI environment. Specifically if ALSA has a MIDI device open it is not available to Java.
Conversely if Java has a device open it is unavailable to ALSA. There are no simple solutions other then to not uses
the two simultaneously. One future possibility is to create new Cadejo node types which would bridge between OSC and MIDI

Virtual Keyboard

A virtual keyboard is provided for quick auditions or for those situations where MIDI may not be a available.

Technically the virtual keyboard is a parent node to the scene object and any open MIDI devices is the parent node to the
virtual keyboard.

[midi] --> [virtual-keyboard] --> [scene]

The virtual keyboard is intended for quick auditions and not as a serious performance tool

Tuning Tables

A tuning table is a map between MIDI key-number and frequency. This mapping is arbitrary and it is possible to establish keys running backwards or to have several alternate scales in the same table, each occupying a separate range of keys.

There are two basic scale types; just and equal tempered. The standard Western 12 note scale is an equal tempered scale. Each scene object automatically contains a tuning table named ":eq-12" for the standard scale.

Tuning tables may have any name so long as they are unique. The convention for equal tempered scales is :eq-n where n is the number of notes per octave.

In order to use alternate tunings select either an available just scale or the number of notes per octave for an equal tempered scale and click the appropriate "add" button. A reasonable default name is automatically suggested but may be changed, if desired, prior to adding the new table. Once a table has been added to the registry it still must be selected for use. This is done by setting the tuning-table property at either the channel or performacne level. In short adding a table to the registry does not automatically select it for use. What it does do is make the table visible to channel and performance nodes.

The current scale registry interface is functional but a bit clunky and may be updated in the future.
