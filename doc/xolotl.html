Overview

Xolotl (pronounced "szho-low-till") is a "recombinant" sequencer and MIDI pattern generator.
Rhythm and pitch patterns are specified separately. If the patterns have
the same length they will always produce the same sequence of events. If
they have different lengths then each time through the rhythm pattern the
pitch pattern will start at a different location, thus the term
recombinant. There are two independent sequences (A and B) which may
operate on different MIDI channels. In addition to rhythm and pitch
patterns each sequence also contains hold time, velocity and 2 MIDI
controller patterns. A sequence may transmit a MIDI program change when it
starts.

Elements in the pitch and velocity patterns may be selected either in
sequence, at random or by the output of a linear-feedback shift register.

There is a bank of 128 xolotl programs which may be recalled either by
external MIDI program change or by clicking on the bank list. Xolotl
programs may be chained together, after a specified number of repeats one
xolotl program may jump to another program.

Clocks and Tempo

The two sequences A and B share a common clock source. By default an
internal clock is used. Provisions for using an external MIDI clock are
provided but have not been tested. One caveat when using an external clock
is that you must tell Xolotl what the tempo is. The clock resolution is 24
ticks per quarter note. The internal clock is global and all instances of
cadejo/Xolotl use the same clock. Tempo may be set between 20 and 300 BPM.

Squences A and B

Each instance of Xolotl contains two independent sequences; 'A' and 'B'.
With exception of the clock source, tempo and current program, these two
sequences are completely independent. The parameters for A and B are nearly
identical. The only exception is that sequence A has parameters related to
jumping to another Xolotl program after a predetermined number of iterations.

MIDI Channels

Each sequence has independent MIDI input and output channels.

MIDI input is used for three purposes:

  • Program Changes
    External program changes may be used to select a Xolotl program.

    A and B may have different input channels. Program changes on either input channel will change the program for both A and B.

  • Transposition
    MIDI key events may be used to transpose the sequencer. For example if a sequencer is playing a major arpeggio, press a C key causes a c-major arpeggio, pressing G produces a G arpeggio etc.

  • Gating sequencer output
    MIDI key events may be set to turn sequencer output on or off.

MIDI output channel sets the channel on which the sequence transmits events.

Note the input and output channels should not be the same as this may set up a feedback situation. A small warning message appears whenever the input and output channels are identical.

Options

  • Enable - enable/disable sequence

  • Key Reset
    If selected every time an initial key is played from an external source a MIDI reset event is generated. This has the effect of starting the sequence over. A rest is only generated on an initial key press.

  • Key Gate
    If selected the sequencer will only produce events if a key is being held down on the input channel.

  • Key Track
    If selected the sequencer pitch pattern is transposed by incoming key events. If key track is not selected a reasonable transpose value should be set.

  • Transpose
    Transposition amount in MIDI key numbers. A transpose value of 0 corresponds with middle C, key-number 60.

  • Program
    MIDI program number to transmit when sequence starts. A value of -1 turns this feature off.

    This value should not be confused with the Xolotl program number. Briefly the Xolotl program number sets the current configuration of Xolotl. This program value causes a program change event to be transmitted by Xolotl to select a synth patch.

  • Mon
    Selecting the mon checkbox enables a debug monitor. Selected information is printed to the terminal whenever the sequencer is stepped. There is no monitor output while the sequencer is running.

Rhythm Pattern


The rhythm pattern sets the master rhythmic motif and is repeated endlessly
unless steps are taken otherwise. The rhythm may be specified either
numerically or symbolically. If number are used they must be positive
integers where 24 indicates a quarter note. The pattern '24 12 12'
indicates a quarter note followed by two eighth notes. A more friendly
method is to specify the rhythm symbolically. The following basic symbols are
recognized: W, H, Q, E, S and T for whole, half, quarter, eighth, sixteenth
and thirty-second note respectively.

Dotted times are indicated by placing a period to the right of the root
time. thus 'H.' indicates a dotted half note. Dotted notes are not
available for thirty-second notes.

Triplets are indicated by a 't' to the right of the root time. 'QT'
indicates a quarter note triplet.

All of the pattern editors actively monitor their contents for
validity. When invalid text is entered the background switches to an
alarming color and a descriptive message appears on the status line. The
internal sequencer state is updated automatically as soon as the pattern
text is valid.

In general pattern text is not case sensitive and white space is used to
separate values.

The sequence A pattern contains two controls missing with sequence B;
'repeat' and 'jump'. The repeat value sets the number of times
the rhythm pattern is played before switching to another Xolotl
program. If repeat is set to 0 the pattern will play indefinitely.

The jump value indicates which Xolotl program to switch to. If jump is set
to -1 it is disabled and the pattern will repeat indefinitely.

Hold Pattern

The hold pattern sets how each note is articulated in terms of relative
note duration. A hold value of 1.00 holds a note for 100 percent of it's
duration. Values greater then 1.0 sustain notes over each other while
values less then 1.0 cause a staccato articulation.

The length of the hold pattern interacts with the length of the rhythm
pattern. The default hold pattern is simply the single value '1.00' which
means every note gets it's entire duration. A rhythm pattern 'Q' with
a hold pattern of '1.00' indicates an endless string of quarter notes. If
we change the hold pattern to '1.00   0.50' the resulting pattern is more
like a quarter note followed by an eighth note followed by an eighth rest.

Pitch patterns


Pitches may be specified either numerically or symbolically. Pitch
values are relative to the external key number (if key-track is enabled)
and the transpose amount. A pitch value of 0 indicates that the pitch of
the most recent key pressed is to be used, or if key-track is disabled the
value of transpose. A pitch value of -12 indicates an octave lower etc. By
default the external key number is 60 (middle C) whether a key has been
depressed or not. The sequence '0 2 4 5 7 9 10' indicates an acceding
major scale.

A better way is to use symbolic pitch names. All pitch names have the form
'P[M]O' where P is the root pitch and one of C D E F G A or B, and O is an
octave number and must be one of -4 -3 -2 -1 0 1 2 3 or 4. The pitch 'C0'
corresponds to integer value 0, C-1 to -12 and C1 to +12 etc. The M
component is an optional modifier and may be either 'F' for flat or 'S' for
sharp. Enharmonic values are defined so that 'CF0' is a valid pitch for
C-flat zero.

The special pitch 'R' indicates a rest. Internally rest are indicated by
negative pitch values. The numeric value of R is -1000.

Chords are defined by enclosing a sequence of pitches in square brackets.
The sequence 'C0 D0 E0 F0 G0 A0 B0 C1 [C0 E0 G0]' produces a c-major scale
followed by a major chord. Brackets must be balanced and nested brackets
are not allowed.

There are three ways notes from the pitch pattern are selected.

  • SEQ - notes are used in strict sequence
  • RND - notes are selected randomly
  • SR - notes are selected by the output of a shift-register

As with the hold pattern if the pitch pattern length does not match the
length of the rhythm pattern then the two patterns interact to produce an
evolving sequence of events.

To the left of the pitch pattern and immediately below the transpose slider
is a small box for setting how chords are played. There are two values, a
delay time and a direction mode. If the delay time is greater then 0
all chords are "strummed" with each note being delayed by the indicated
time in milliseconds. The four radio buttons select the direction the strum.

Velocity and Controllers

The velocity and controller patterns are similar to the hold pattern
except are specified as integers between 0 and 127 inclusive. The velocity
pattern has the same selection options as the pitch pattern. The two
controls below the controller pattern editors select the MIDI controller
number. A value of -1 turns the controller off.

Shift Register


The Shift Register is used to produce complex periodic signals. Conceptually a
shift-register (as used by Xolotl) is a linear array of cells, each holding a
binary value of either 0 or 1. At intervals determined by the rhythm-pattern
the cell contents are shifted to the right and a new value is shifted into
the vacated left-most cell. The value shifted into the left is determined
by selecting or "tapping" some of the cells and summing their values. If
the sum is even the new value is 0 otherwise it is 1. A separate "inject"
value may optionally be added to the feedback sum.

At any single time the 'value' of the register is taken as the weighted
sum of the cells from left to right (MSB on the right). The exact sequence
of values produced by the register is largely determined by the selection of
taps and the initial or "seed" value. The 'period' of the register is the
number of values it produces in sequence before returning to the initial
state. For a register of n-bits the maximum period is 2^n. The shift
register used by Xolotl has a maximum length of 12 bits so the upper limit
of period length is 4096 steps. The reality however is that most
tap combinations do not produce the maximum period. These shorter patterns
are actually more interesting as longer periods tend to sound random.

The shift-register output may be used to select notes or chords from the
pitch pattern or velocity values from the velocity pattern. Since the range
of values an n-bit register may produce (between 0 and 2^n-1) is likely
larger then the number of elements to be selected, the register value is
taken modulo to the pattern length. Changes made to a pattern, either by
insertion, deletion or rearrangement will produce different sequences.

There are three sets of shift-register controls in the form of three rows
of radio buttons. The top row selects which stages are to be used for
feedback.

The middle row selects the initial or "seed" value.

The bottom row is a mask applied to the registers output. You may shorten
the register by unselected all buttons to the right of the desired
length. The mask does not effect the tap values so any selected tap to the
right may still influence the register's behavior. The mask may also be
used to eliminate selected bits from contributing to the output.

The final shift-register control is the inject button. When selected
inject adds one to the feedback count. Usually this results in a different
pattern. It may also be used to prevent the register from freezing up. If
the register ever contains all zeros then it is frozen. Using inject
ensures at least one cell will contain a one.

Program Bank

The Xolotl program bank stores up to 128 sequencer
configurations which may be saved to an external file.
The three components below the bank list are for storing
the current program into the bank. The spinner selects
the destination slot number between 0 and 127.
