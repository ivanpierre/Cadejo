At the top left-hand side of Performance frames are three buttons for accessing various
sub panels. These buttons are MIDI, CTRL, and Edit.

The MIDI tab is selected in the image below and is identical to the Channel MIDI tab.
The only difference is that now the properties shown are at the Performance level.

The CTRL tab is for editing MIDI continuous controller properties. The content of this tab is
similar to the MIDI tab. For each controller the instrument responds to there are corresponding
curve, scale, bias and inherit widgets.

The Edit tab is for editing instrument programs or "patches". The exact content of this tab is
determined by the specific instrument.

Program Banks


Program banks hold 128 programs numbered between 0 and 127 and are always visible on the left side of performance frames.

At the bottom of Performance frames are where bank related controls are found. The numeric display,
cluster of arrow buttons, and "store" button are for saving the current patch into the bank.
The number displays the bank slot where the program is to be saved. This value is automatically changed
whenever a new program is selected (either manually or by MIDI program change). The four arrow buttons are
for selecting the store location. The single arrows increment or decrement the slot number by 1. The double
arrows change the slot number by 8. Click the "store" button to save the current program into the selected slot.

Immediately to the right of the "store" button is a large area to display current program name.
The name is truncated if it is too large and is always converted to upper case. Program names are mixed
case so this display may not exactly match the actual program name.

Whenever the current program data has been altered, the word 'modified' appears above the program name.
Once the program has been saved the word modified disappears. The code responsible for displaying modified runs on a
low-priority thread and it may not appear/disappear immediately

The buttons to the right of the program name are for bank initialization, Open and Save bank data, and undo/redo of
bank changes. The undo/redo feature is for changes to the bank only and not for individual program changes.

Program banks are saved as text files (Clojure code) and have filenames:  name.xxx_bank    where 'xxx' is the Cadejo
instrument's name.

Progressive Program Mode

Progressive Program Mode allows multiple programs to be used simultaneously.
Progressive mode is active whenever the progressive mode display shows a number greater then 1.
Use the up and down arrow buttons to change the progressive mode count.

When a program change is received and the progressive mode count is greater then 1, each voice
receives a different program from the program bank. This is easier to describe by example.

Assume the progressive value is 2 and there are 8 polyphonic voices (the default).
In this situation whenever a MIDI program change occurs for program n, the first voice
receives program n, the second voice receives program n+1, the third voice receives program n
the fourth voice program n+1. This alternation between program n and n+1 continues for all 8 voices.
If the progressive count is 3 the voices revives programs n, n+1 and n+2 in a cycle etcetera.

Progressive Mode Caveats:

  • Progressive mode works best with rotate keymode, and not at all with mono keymodes. See keymodes
  • Program editing is disabled when progressive mode is active. Which program would we be editing?
  • Currently (24-Sep-15) progressive mode is only activated by external MIDI program change.
    Clicking an entry in the program bank list does not work with progressive mode. Hopefully this will be fixed soon.

Historical Perspective:

Oberheim released the "Synthesizer Expander Module" or SEM in 1974. The SEM was a small self-contained monophonic synthesizer
packaged as a module and without keyboard. By 1975 they started strapping 2, 4 or 8 SEM modules together in a single package
together with an added keyboard to produce some of the first polyphonic synthesizers. Each voice was an analog instrument with
it's own set of panel controls. With some work these separate components could be adjusted to sound nearly identical but
never exactly identical. Each key-press had a subtly different sound. Of course the instrument could also be patched to have
drastically different sounds on each key-press as well.

CTRL tab

The CTRL tab is for setting MIDI continuous controller properties. It is similar in
layout to the MIDI tab with an edit area for each controller. Each instrument defines
exactly which controllers are available.

Controller values are normalized to a range between 0.0 and +1.0. The final value is
given by: 

 cc value -->[norm]-->[mapping curve]-->[* scale]-->[+ bias]-->

Instrument Editor


The performance Edit tab displays the instrument editor. The editor is not created until
the Edit tab is clicked. The first time the edit tab is clicked it may take several seconds
to appear while it is being created. Subsequent clicks on the edit tab will be more
responsive.

The buttons to the left are common to all editors:

  • Init - Initialize program
  • Random - Create random program
  • Open - Open single program file. Single program filenames have the form 'xxx.iii' where iii is the instrument name.
  • Save - Write single program file
  • Copy - Copy current program to clipboard
  • Paste - Paste clipboard to current program
  • Help - Instrument specific help

The buttons to the right are instrument specific. Each instrument will define it's own set of
editor tabs. All instruments have the 'common' tab, which is where the program name is set.
The image above shows the editor for the Combo organ which only has two tabs; "Common" and "Edit",
with the Common tab selected.

The area immediately below the program name field shows the Clojure code which defines the
current program. In future releases this area may be used for some other purpose.
