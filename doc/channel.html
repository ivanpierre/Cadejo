Channel nodes are an intermediate stage between incoming MIDI messages and final instruments (performances).
Their primary duty is to aggregate one or more instruments in a layer.

Adding Instruments

To add an instrument click one of the large available instrument buttons and an 'add instrument' dialog will appear.

Key modes are described here.

Output bus, Sets the SuperCollider audio bus used as the main output for this instrument. Defaults to 0, the first sound card.

Voice Count, Sets the number of voices for polyphonic keymodes. Default 8.

Controllers, Sets MIDI controller numbers. Each instrument type will have it's own set of controllers. Controllers common to most instruments are cc1 for mod wheel and cc7 for volume. All controllers have reasonable defaults but may be freely reassigned.

Instrument ID, On any given channel each instrument must have an unique id. A valid id is automatically generated but may be changed if desired.


Once an instrument has been added a new button appears in the 'active instrument'
area. Clicking these buttons pops up a frame for the instrument's performance node.

You can try to shift-click on an instrument to remove it, but this feature is buggy

Key Modes

There are four possible keymodes:

  • Exclusive
  • Mono
  • Poly
  • Rotate

Not all keymodes are available for all instruments.

Exclusive is a monophonic mode where only the first keydown is articulated. As long as each
note is played staccato the note will sound. For legato passages however only the first note played will sound.
An instrument in exclusive mode is intended for use in a layer with some other instrument to provide an
alternative sound for the first note of a legato passage. A typical example would be an organ key click.

Mono is a generalized monophonic mode. If two or more keys are pressed only the last key down
produces a note. Previous keys are not forgotten however making it possible to play monophonic trills.
Mono mode is not the same as a single-voice polyphonic mode.

Poly is a polyphonic mode which attempts to use voices in order. On the first key down voice 1 is
is used. If the first key is released and pressed a 2nd time, voice 1 will again be used. If the first key is held
while a 2nd key is pressed, voice 2 is used for the 2nd key, and so on. If the number of keys down exceeds the voice
count then the most stale (IE the voice which has been holding the longest) is reallocated for the new note.

Rotate is another polyphonic mode only this time voices are allocated in rotation. If the same key
is repeatedly pressed and released a new voice is used for each key press. Rotate mode uses a similar
"voice stealing" scheme as ploy mode if the voice count is exceeded.

The difference between Poly and Rotate modes can be subtle. Where the difference becomes apparent is when the
program-bank is in progressive mode. Normally when an instrument receives a MIDI program change all voices are
updated to the same "patch" and thus produce the same sound. In progressive mode each voice may have a completely
different patch. Each new key press may produce a different sound. The Rotate keymode was specifically developed for
this purpose. More details on progressive program changes may found in the program bank documentation.
