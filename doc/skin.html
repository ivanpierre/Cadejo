Cadejo may be customized by selecting one of several substance skins. There are two ways to do this.
The easiest way is to click the "skin" button at the top of most Cadejo frames. The advantage of this method is
that it gives you instant feedback. The disadvantage is that the changes are not permanent. A further disadvantage
is that not all graphical elements will be updated. Cadejo uses two graphic libraries: swing and sgwr. Swing elements
will be updated to match the new skin but any existing sgwr drawings will not be updated. Any future sgwr drawings
will match the new look.

A more permanent method is to set the skin in a configuration file.

Currently configuration is hard-coded in "cadejo/config.clj". The ability to
load configuration files at run time is planned for the future.

There are two configuration values related to the look of cadejo; "initial-skin" and "displaybar-style"
The value for initial-skin must be a String and must exactly match one of the available skins. config.clj
contains a list of possible skin names.

displaybar-style may have one of the fowling values (nil :basic :sixteen :matrix :twilight)
A displaybar is a graphic element defined by the sgwr library used to display alpha-numeric values and there are
four possible styles.


Basic style uses a native sans-serif font and is both flexible and efficient.

Matrix is a 5x7 dot-matrix display. It is flexible in the number of possible characters it may display
but due to the several hundred graphic elements a typical display uses takes a long time to create and update.

Sixteen style emulates a 16-segment display

Twilight uses a set of icons to simulate a matrix display using the twilight color scheme. It is quicker
then the matrix style but has fixed colors which may or may not work with non twilight skins.

If the config displaybar-style is set to nil, the skin will supply a default style. Any other value
explicitly overrides these defaults.
